Trytet Documentation

WebAssembly sandbox engine for AI agent tool use. Executes LLM-generated code in fuel-bounded Wasm sandboxes.

Quickstart

curl -sL https://github.com/bneb/trytet/releases/latest/download/tet-darwin-aarch64.tar.gz | tar xz
sudo mv tet /usr/local/bin/
tet mcp      # Start MCP server for Claude Desktop / Cursor
tet up       # Start API server on port 3000

Installation

  • macOS (Apple Silicon)
  • Linux (ARM64)
  • Docker
  • Build from source

macOS

curl -sL https://github.com/bneb/trytet/releases/latest/download/tet-darwin-aarch64.tar.gz | tar xz

Linux

curl -sL https://github.com/bneb/trytet/releases/latest/download/tet-linux-aarch64.tar.gz | tar xz
sudo mv tet /usr/local/bin/

Docker

docker build -t trytet-api:latest .
docker run -p 3000:3000 trytet-api:latest

Build from source

git clone https://github.com/bneb/trytet
cd trytet
cargo build --release --bin tet

Requires Rust 1.80+, cmake, clang, protobuf-compiler.

MCP Server

The Model Context Protocol server exposes 5 sandboxed tools:

  • trytet_js_evaluator — Execute JavaScript with fuel and memory limits
  • trytet_regex_evaluator — Run regex patterns safely (ReDoS-protected)
  • trytet_jmespath_evaluator — Query JSON with JMESPath
  • trytet_scraper — Fetch URL content
  • trytet_structured_data — Filter and sort JSON arrays

Configure Claude Desktop by adding to claude_desktop_config.json:

{
  "mcpServers": {
    "trytet": {
      "command": "tet",
      "args": ["mcp"]
    }
  }
}

API Reference

  • POST /v1/tet/execute — Execute a Wasm payload
  • POST /v1/tet/snapshot/{id} — Snapshot agent state
  • POST /v1/tet/fork/{snapshot_id} — Fork from snapshot
  • POST /v1/mcp — JSON-RPC endpoint for MCP
  • GET /v1/swarm/metrics — Benchmark report
  • GET /health — Health check

SDKs

TypeScript

npm install @trytet/client
import { TrytetClient } from '@trytet/client';
const client = new TrytetClient();
const result = await client.execute({ payload: wasmBytes, allocated_fuel: 1_000_000 });

Python

pip install trytet-client
from trytet_client import TrytetClient, TetExecutionRequest
async with TrytetClient() as client:
    result = await client.execute(TetExecutionRequest(payload=wasm_bytes, allocated_fuel=1_000_000))

Building Cartridges

Cartridges are Wasm components that implement the cartridge-v1 interface. See the cartridge guide for the interface and architecture.

CLI Reference

  • tet up — Start API server
  • tet mcp — Start MCP server
  • tet ps — List running agents
  • tet metrics — Run benchmarks
  • tet publish <file> --tag <name> — Publish cartridge
  • tet search <query> — Search cartridge registry
  • tet validate <file> — Validate cartridge
  • tet init [name] — Scaffold agent project
  • tet key-create --label <name> — Create API key
  • tet key-list — List API keys
  • tet key-revoke <prefix> — Revoke API key
